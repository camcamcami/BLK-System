# BLK-017 — BLK-test MCP Disabled Transport Skeleton

**Status:** Active disabled transport contract
**Scope:** BLK-SYSTEM-011 dependency-free disabled stdio transport descriptors, startup refusal, non-executing handshake/lifecycle probes, and descriptor-only fixed tool registry metadata.

---

## 1. Purpose

BLK-017 makes the BLK-SYSTEM-011 disabled BLK-test MCP transport skeleton discoverable as active doctrine. It records the current safe contract for future transport work while keeping live BLK-test MCP authorization out of scope.

This doctrine exists because BLK-008 describes a future target-state MCP server, BLK-015/016 describe disabled request/adapter fixture contracts, and BLK-SYSTEM-011 adds a narrower local transport boundary harness. BLK-SYSTEM-011 is disabled by default, stdio-only, and limited to deterministic metadata/probe work.

---

## 2. Current authority boundary

BLK-SYSTEM-011 is a disabled transport skeleton only. It does not authorize live BLK-test MCP and does not authorize live MCP client/server startup. The transport boundary is stdio-only metadata/probe work; no HTTP, WebSocket, TCP, UDP, daemon, listener, MCP SDK startup, or live client/server runtime is authorized.

The contract preserves BLK-001 domain separation:

- BLK-test remains a future physics oracle boundary, not a scope planner or source mutator.
- BLK-pipe remains the blast shield/forge for deterministic source mutation and Git staging.
- RTM remains an offline aggregator/ledger and is not generated by BLK-SYSTEM-011.
- Protected BLK-req vault bodies remain outside Sprint 011 scope.

BLK-SYSTEM-011 does not authorize authoritative BEO publication, does not authorize RTM generation, does not authorize RTM drift rejection authority, does not read protected BLK-req vault bodies, must not mutate source, and must not grant arbitrary shell.

## 3. Disabled startup preflight shape

The implementation module is `python/blk_test_mcp_disabled_transport.py`. The disabled startup preflight shape is produced by:

- `build_disabled_transport_descriptor(...)`
- `evaluate_disabled_transport_startup(...)`

Required authority behavior:

- `transport` must be `stdio`.
- Default startup is disabled by default.
- `enabled=True` is rejected instead of enabling transport.
- Approval-looking records remain unsupported and blocked; Sprint 013 owns approval/source-evidence authorization mechanics.
- No server starts, no client starts, no subprocess is spawned, no network is called, and no fixed-tool tests execute.

## 5. Fixed tool registry descriptor-only rule

BLK-SYSTEM-011 may define fixed tool names as metadata only. The descriptor helper is:

- `fixed_tool_registry_descriptor()`

Allowed descriptor-only names are:

- `run_ast_validation`
- `run_ipc_race_test`
- `run_svg_export_purity_test`
- `run_architecture_lint`

Each registry entry remains `DESCRIPTOR_ONLY`; `executor_available` is false; source mutation, authoritative BEO publication, RTM generation, and active-vault reads are disabled.

The execution evaluator is:

- `evaluate_disabled_tool_execution(...)`

It always blocks tool execution. BLK-SYSTEM-011 does not execute fixed-tool tests and must not grant arbitrary shell, dynamic command execution, source-write, staging, commit, autofix, authoritative BEO publication, RTM generation, RTM drift rejection authority, or protected active-vault read authority.
